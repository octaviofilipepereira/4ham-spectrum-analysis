📡 Beacon Analysis — NCDXF/IARU

The NCDXF International Beacon Network transmits every 10 seconds on 5 bands (20/17/15/12/10 m).
Each beacon sends its callsign in CW at 22 WPM, followed by 1 s dashes at 100 W, 10 W, 1 W and 100 mW. Use this mode to assess propagation from 18 fixed stations around the world.

⚠️Scheduler takes over scan engine. Band buttons disabled while active.

      Install Direwolf — Enable APRS

      Direwolf (the software TNC used for APRS packet decoding) is not installed on this server. To install it without re-running the full 4ham installer, run the helper script below.

      Run this command in a terminal on the server where 4ham is installed, follow the prompts, then reload this page.

      The script must be run from inside the 4ham project folder (e.g. cd ~/4ham-spectrum-analysis first).
      What does the script do?
      • Detects your Linux distribution (Debian/Ubuntu/Mint/Raspbian).
      • Installs Direwolf via apt-get install direwolf (asks for your sudo password).
      • Creates a default config/direwolf.conf if missing (KISS TCP on port 8001).
      • Prints next steps to enable APRS in this Admin Config panel.

      Install gr-lora_sdr — Enable LoRa APRS

      The gr-lora_sdr GNU Radio Out-Of-Tree module (used to decode LoRa frames on 868.000 MHz EU SRD / also 433.775 MHz / 70 cm) is not installed on this server. To install it without re-running the full 4ham installer, run the helper script below.

      Run this command in a terminal on the server where 4ham is installed, follow the prompts, then reload this page.

      The script must be run from inside the 4ham project folder (e.g. cd ~/4ham-spectrum-analysis first). The build typically takes 5–15 minutes depending on hardware.
      What does the script do?
      • Installs GNU Radio + build dependencies via apt-get.
      • Clones tapparelj/gr-lora_sdr into /opt/gr-lora_sdr.
      • Builds and installs it with CMake to /usr/local.
      • Verifies the Python module gnuradio.lora_sdr imports successfully.
      • Prints next steps to enable LoRa APRS in this Admin Config panel.

        SSB Voice Transcription (ASR) checking…

        When active, detected SSB voice signals are automatically transcribed and the text is shown in the live notification toast and stored in the event log (raw field). Transcription runs in a background thread and does not affect scanning. On slower hardware (e.g. Raspberry Pi) each 15 s capture takes 2–5 s to process. Requires openai-whisper — install it via the installer or manually: pip install openai-whisper. You can enable/disable this at any time without restarting.

        APRS Packet Decoding (Direwolf) checking…

        When active, the backend starts Direwolf and connects to its KISS TCP port to decode APRS packets on 144.800 MHz (VHF). Decoded packets appear in the live event stream and are stored in the event log. Requires direwolf — if the badge above shows “Direwolf not installed”, click Install Direwolf… for the one-line command to install it without re-running the full installer.

        APRS-IS Internet Server

        TCP server used for the APRS-IS Internet feed. Default rotate.aprs2.net:14580 is the official DNS rotator (auto-picks the closest healthy server). Use a regional hostname (e.g. euro.aprs2.net, noam.aprs2.net, asia.aprs2.net) to force a specific continent.

        LoRa APRS Packet Decoding (gr-lora_sdr) checking…

        When active, the backend listens on UDP 127.0.0.1:5687 for decoded LoRa frames forwarded by a gr-lora_sdr flowgraph (868.000 MHz EU SRD by default; 433.775 MHz / 70 cm also supported). Decoded packets appear in the live event stream and are stored in the event log with source=lora_aprs. Requires gr-lora_sdr — if the badge above shows “gr-lora_sdr not installed”, click Install gr-lora_sdr… for the one-line command to install it without re-running the full installer.

        Quick tips

        • Switch band live using the quick-band buttons (160m … 10m) without stopping the scan.
        • Use Logs & Reports → Search & Export Data to filter events and export CSV / JSON / PNG.
        • The Propagation Score (0–100) reflects band quality from confirmed decodes (verified callsign) over the last 60 min — events without a callsign = band occupancy.
        • FT8/FT4 waterfall markers expire after 3 missed decode cycles — this is normal during low-activity periods.
        • Admin settings (SDR device, gain, retention) require admin login via Admin Config.
